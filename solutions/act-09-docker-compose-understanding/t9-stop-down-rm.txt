1. docker compose up -d
Note: This command created and started the web and redis containers in the background (detached mode). Even though the terminal is free, the services are actively running and accessible.

2. docker compose stop
Note: This gracefully halted the processes inside the containers without deleting them. The containers still exist on your disk, preserving their current state/data for a quick restart later.

3. docker compose ps
Note: This listed the status of your services. Since you just ran stop, it confirms that the containers are present but inactive (status would typically show as Exited).

4. docker compose rm -f
Note: This permanently deleted the stopped container instances from your system. The space they occupied is now free, but the Docker network created for this project still exists in the background.

5. docker compose down
Note: Since the containers were already removed, this final step cleaned up the infrastructure by removing the lab3_default network. Your environment is now completely clean, exactly as it was before you started.
